<h1>

</h1>
<p><p>Given an array of integers <code>nums</code> and an integer <code>target</code>, return <em>indices of the two numbers such that they add up to <code>target</code></em>.</p>  <p>You may assume that each input would have <strong><em>exactly</em> one solution</strong><br>, and you may not use the <em>same</em> element twice.</p>  <p>You can return the answer in any order.</p>  <p> </p> <p><strong class="example">Example 1:</strong><br></p>  <pre> <strong>Input:</strong><br> nums = [2,7,11,15], target = 9 <strong>Output:</strong><br> [0,1] <strong>Explanation:</strong><br> Because nums[0] + nums[1] == 9, we return [0, 1]. </pre>  <p><strong class="example">Example 2:</strong><br></p>  <pre> <strong>Input:</strong><br> nums = [3,2,4], target = 6 <strong>Output:</strong><br> [1,2] </pre>  <p><strong class="example">Example 3:</strong><br></p>  <pre> <strong>Input:</strong><br> nums = [3,3], target = 6 <strong>Output:</strong><br> [0,1] </pre>  <p> </p> <p><strong>Constraints:</strong><br></p>  <ul> 	<li><code>2 <= nums.length <= 10<sup>4</sup></code></li> 	<li><code>-10<sup>9</sup> <= nums[i] <= 10<sup>9</sup></code></li> 	<li><code>-10<sup>9</sup> <= target <= 10<sup>9</sup></code></li> 	<li><strong>Only one valid answer exists.</strong><br></li> </ul>  <p> </p> <strong>Follow-up: </strong><br>Can you come up with an algorithm that is less than <code>O(n<sup>2</sup>)</code><font face="monospace"> </font>time complexity?</p>
